# Naming Conventions


In the project we lousely follow Google C/C++ style, that is we use the following naming rules.

| | |
|:------------------------------------:|:-------------------------------|
| Type names (structs, classes, enums) | `PascalCase`                   |
| Functions                            | `camelCase` *                  |
| Variables                            | `snake_case`                   |
| Constants                            | `kPascalCase` (use `k` prefix) |
| Macros                               | `SCREAMING_SNAKE`              |
| Namespaces                           | `snake_case`                   |

*We are using clang-type function names.
